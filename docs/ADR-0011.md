# ADR-0011: Sovereign Edge Vault V1 Honest Scope

## Status
Accepted

## Context
The previous `SovereignDataVault` surface implied nation-held keys, on-prem deployment,
and data residency controls that the codebase did not actually implement. That overstated
the platform state and materially inflated the score for challenge `#14`.

Wave E narrows the claim to an honest local-only release:

- browser-side PBKDF2 key derivation
- browser-side AES-GCM export encryption
- consent-gated export and import
- local audit logging

This is enough to move the implementation from a UI mockup to a real edge-vault v1, but
not enough to claim full Indigenous data sovereignty infrastructure.

## Decision
The feature is positioned as `Edge Vault V1` / `OCAP-ready edge vault v1`, not a completed
sovereign hosting platform.

The implementation deliberately avoids claiming:

- nation-held production key custody
- cloud residency enforcement
- append-only remote attestation
- on-prem execution or Dockerized deployment
- multi-device sync or isolation guarantees

The score for `#14` is capped at `3.2 / 5` until those controls exist outside the browser.

## Technical Details
- Exported vault snapshots use `PBKDF2(SHA-256, 210000 iterations)` to derive an AES-256-GCM key.
- The export envelope carries only non-secret metadata plus `salt`, `iv`, and `ciphertext`.
- The visible fingerprint is derived from hashed envelope metadata, not from fake key material.
- Consent state and the audit trail are stored locally in the browser for v1.

## Consequences
Positive:

- The vault now performs real cryptographic work locally.
- Export and import behavior is testable and deterministic.
- Product copy now matches the actual implementation boundary.

Negative:

- The implementation is still single-device and local-first.
- Nations must manage passphrase custody outside the application.
- There is no proof of residency or infrastructure ownership beyond the local workflow.

## One-Week Review Prerequisites
- On-prem runtime packaging choice: container bundle, local desktop runtime, or managed appliance.
- External key custody model: nation-held HSM, managed KMS with customer-held root, or split-custody design.
- Residency enforcement boundary: local-only, community network, or attested private cloud.
- Tamper-evident audit design: append-only local ledger vs remote attestation service.
- Operator runbook: backup, passphrase rotation, recovery, and custody transfer procedures.
