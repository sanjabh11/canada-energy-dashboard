# CEIP Comet Phase F Buyer-Evidence Runbook

**Status:** canonical operator runbook for Comet-assisted Phase F rehearsal and real buyer-evidence collection.  
**Last updated:** 2026-06-02.  
**Boundary:** this runbook may create intake scaffolding and retained-artifact records, but it must not create buyer proof, reviewer acceptance, commercial commitments, or market-confidence movement unless real anonymized buyer-supplied evidence exists.

## 2. What Comet May And May Not Create

| Item | Allowed? | Rule |
|---|---:|---|
| Test-only outreach rehearsal rows | Yes | Must say `REHEARSAL ONLY`, `commercial_commitment_status=none`, and `confidence_delta=0`. |
| Real anonymized outreach rows | Yes | Only after a real buyer reply exists; do not retain names, emails, phone numbers, addresses, account IDs, meter IDs, or secrets. |
| Intake packet folders | Yes | They are starter scaffolds only and must not be described as accepted evidence. |
| Retained redacted artifacts | Yes | Only from already-redacted, buyer-approved, text-inspectable extracts with SHA-256 hashes. |
| Reviewer acceptance | No, unless real | Must come from a real reviewer and must be retained in redacted text. |
| Commercial commitment | No, unless real | Only `design_partner_signed`, `paid_pilot`, `purchase_order`, or `letter_of_intent` count, and the retained artifact must support the claim. |
| `confidence_delta > 0` | No, unless real | Positive confidence movement requires accepted buyer-supplied retained artifacts. |
| 95% market-confidence claim | No, unless hard gate passes | The hard gate must pass with `--require-95 --evidence-root`. |

Never invent files such as `REHEARSAL-NOTICE.md` or `REHEARSAL-ONLY.flag`. If those files are needed for a future workflow, add them through the repo first and validate them. Until then, use only the files generated by the scripts below.

## 3. Manual Rehearsal Workflow

Use this when the app is not released enough to collect real replies, or when Comet needs to practice the exact workflow. This does not move confidence.

```bash
pnpm run create:phase-f-evidence-workspace -- --output-dir /tmp/ceip-phase-f-rehearsal --record-date 2026-06-02
```

Append clearly simulated outreach rows. Use the canonical appender so the route, lane, and proof-pack identity remain valid:

```bash
pnpm run append:outreach-response-log-row -- \
  --log-file /tmp/ceip-phase-f-rehearsal/outreach/outreach-response-log.csv \
  --activity-date 2026-06-02 \
  --channel manual \
  --target-label rehearsal_utility_planner_001 \
  --route /utility-demand-forecast \
  --rating 4.5 \
  --variant-id phase_f_rehearsal_utility_forecast \
  --reply-status data_offered \
  --response-summary "REHEARSAL ONLY: simulated anonymized utility planning reply for workflow practice; no real buyer interaction." \
  --pain-signal "rehearsal load growth planning and forecast defensibility" \
  --requested-input "rehearsal redacted load history sample" \
  --reviewer-role "utility planning reviewer" \
  --commercial-commitment-status none \
  --next-action "create intake packet for rehearsal only" \
  --pilot-evidence-register-action create_intake_packet \
  --notes "test-only rehearsal row; not buyer evidence"
```

Repeat with one TIER or credit route and one billing or utility-security route. The current minimum rehearsal mix is:

| Route | Proof-pack lane | Example target label |
|---|---|---|
| `/utility-demand-forecast` | utility forecast | `rehearsal_utility_planner_001` |
| `/roi-calculator` | TIER CFO | `rehearsal_industrial_cfo_001` |
| `/utility-security` | security alternative to shadow billing | `rehearsal_security_reviewer_001` |

Validate the log, plan intake, and generate packets:

```bash
pnpm run validate:outreach-response-log -- /tmp/ceip-phase-f-rehearsal/outreach/outreach-response-log.csv
pnpm run plan:outreach-intake -- /tmp/ceip-phase-f-rehearsal/outreach/outreach-response-log.csv
pnpm run create:outreach-intake-packets -- \
  --log-file /tmp/ceip-phase-f-rehearsal/outreach/outreach-response-log.csv \
  --output-dir /tmp/ceip-phase-f-rehearsal/outreach-intake-packets \
  --force
```

Report the rehearsal workspace:

```bash
pnpm run report:phase-f-evidence-workspace -- --workspace-dir /tmp/ceip-phase-f-rehearsal
```

The report must still say `Confidence movement: none`, `Buyer proof created: no`, and `Hard 95% retained-evidence gate: blocked`.

## 4. Real Buyer-Evidence Workflow

Use a different workspace for real evidence so rehearsal files cannot be confused with production evidence:

```bash
pnpm run create:phase-f-evidence-workspace -- --output-dir /tmp/ceip-phase-f-evidence --record-date YYYY-MM-DD
```

After a real outreach reply exists, append an anonymized row:

```bash
pnpm run append:outreach-response-log-row -- \
  --log-file /tmp/ceip-phase-f-evidence/outreach/outreach-response-log.csv \
  --activity-date YYYY-MM-DD \
  --channel linkedin \
  --target-label utility_planner_001 \
  --route /utility-demand-forecast \
  --rating 4.5 \
  --variant-id utility_forecast_phase_f \
  --reply-status data_offered \
  --response-summary "Buyer asked for a bounded forecast sample from redacted load history." \
  --pain-signal "load growth planning and forecast defensibility" \
  --requested-input "redacted interval load history" \
  --reviewer-role "utility planning reviewer" \
  --commercial-commitment-status none \
  --next-action "create intake packet" \
  --pilot-evidence-register-action create_intake_packet
```

Plan and create intake packets:

```bash
pnpm run plan:outreach-intake -- /tmp/ceip-phase-f-evidence/outreach/outreach-response-log.csv
pnpm run create:outreach-intake-packets -- \
  --log-file /tmp/ceip-phase-f-evidence/outreach/outreach-response-log.csv \
  --output-dir /tmp/ceip-phase-f-evidence/outreach-intake-packets
```

Create a retained redacted artifact only after the buyer-approved extract exists:

```bash
pnpm run prepare:pilot-evidence-artifact -- \
  --evidence-root /tmp/ceip-phase-f-evidence/phase-f-minimum-intake/utility-demand-forecast/redacted-artifacts \
  --artifact-file redacted-utility-forecast.md \
  --route /utility-demand-forecast \
  --proof-pack-id utility_forecast_planning_pack \
  --record-date YYYY-MM-DD \
  --pii-screen-result redacted \
  --buyer-data-coverage-pct 90 \
  --time-to-artifact-hours 36 \
  --reviewer-role "utility planning reviewer" \
  --reviewer-acceptance accepted \
  --reviewer-feedback-status complete \
  --day-14-decision proceed \
  --commercial-commitment-status letter_of_intent \
  --commercial-commitment-evidence "redacted LOI evidence retained in text appendix" \
  --claim-boundary "Buyer-supplied redacted planning support only; no production utility onboarding claim." \
  --do-not-claim "Do not claim utility approval, live telemetry, guaranteed savings, or control-room use." \
  --diagnostic "MAE 12.4 MW; MAPE 3.8%; RMSE 18.6 MW; persistence MAE 21.3 MW; seasonal-naive MAE 19.9 MW; rolling-origin split count 4; interval coverage 91.2%; CEIP champion vs seasonal-naive challenger."
```

Use the printed `sha256` reference to update the selected register:

```bash
pnpm run update:pilot-evidence-register-row -- \
  --register-file /tmp/ceip-phase-f-evidence/phase-f-minimum-intake/phase-f-minimum-register-starter.csv \
  --evidence-root /tmp/ceip-phase-f-evidence/phase-f-minimum-intake \
  --artifact-root /tmp/ceip-phase-f-evidence/phase-f-minimum-intake/utility-demand-forecast/redacted-artifacts \
  --evidence-file-reference "redacted-utility-forecast.md#sha256=<hash-from-helper>" \
  --confidence-delta 0.3 \
  --output-file /tmp/ceip-phase-f-evidence/phase-f-minimum-register-updated.csv
```

Rerun the workspace report against the updated register:

```bash
pnpm run report:phase-f-evidence-workspace -- \
  --workspace-dir /tmp/ceip-phase-f-evidence \
  --register-file /tmp/ceip-phase-f-evidence/phase-f-minimum-register-updated.csv
```

Only after all three minimum lanes are accepted and hashed, run the hard gate:

```bash
pnpm run validate:pilot-evidence -- \
  /tmp/ceip-phase-f-evidence/phase-f-minimum-register-updated.csv \
  --require-95 \
  --evidence-root /tmp/ceip-phase-f-evidence/phase-f-minimum-intake
```

## 5. Manual Inputs Required Before The Hard Gate Can Pass

| Requirement | Accepted evidence |
|---|---|
| Utility forecast proof pack | Accepted buyer-supplied utility forecast row with MAE, MAPE, RMSE, persistence baseline, seasonal-naive baseline, rolling-origin split count, interval coverage, and champion/challenger diagnostics. |
| TIER or credit proof pack | Accepted buyer-supplied TIER CFO or credit-banking row. |
| Shadow-billing or utility-security proof pack | Accepted buyer-supplied billing or security row. |
| Retained redacted artifacts | Text-inspectable retained files with locally recomputed SHA-256 references. |
| Reviewer acceptance | `reviewer_acceptance` accepted/approved/signed, `reviewer_feedback_status` complete/accepted/approved/signed, and `day_14_decision=proceed`. |
| Confidence movement | Total accepted `confidence_delta >= 0.9`, with no single row above the validator cap. |
| Fast-delivery proof | At least one accepted buyer proof pack with `time_to_artifact_hours <= 48`; no accepted confidence-moving row above 120 hours. |
| Commercial signal | At least one accepted row with `design_partner_signed`, `paid_pilot`, `purchase_order`, or `letter_of_intent`, supported by retained redacted evidence. |
