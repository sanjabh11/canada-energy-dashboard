## Executive Summary

**Health grade: B-**

The Canada Energy Dashboard (CEIP) is a feature-rich React 18 + TypeScript SPA with a Supabase backend that has undergone significant security hardening, performance optimization, and commercial positioning discipline across multiple sprints. The codebase demonstrates strong claim-boundary enforcement and proof-pack architecture — unusual for a product at this maturity stage. However, several issues prevent a higher grade:

**Top 3 risks:**
1. **Test suite instability** — 17 of 127 test files fail in parallel runs (66 tests), primarily due to jsdom environment issues and slow readiness-report tests timing out. This undermines CI confidence.
2. **Large-file maintainability** — `helpContent.ts` (5,727 LOC), `advancedForecasting.ts` (2,648 LOC), and `UtilityDemandForecastPage.tsx` (2,290 LOC) are gravity wells that resist modification and testing.
3. **Pricing synchronization gap** — `pricingCatalog.ts` is the canonical source but README, Top20, and COMMERCIAL_SOURCE_OF_TRUTH contain hardcoded pricing prose that can drift.

**Top 3 opportunities:**
1. **Scenario workbench substrate** — The sensitivity/uncertainty engine is architecturally sound and now type-safe; completing connector fixture tests would unlock a defensible analytical differentiator.
2. **Buyer-language aliases** — Adding SEO/search aliases for proof-pack routes would expand addressable market without breaking claim guardrails.
3. **Cross-sell bundle maps** — The utility, industrial, municipal, and consultant spines exist as routes but are not explicitly mapped as bundles, leaving revenue on the table.

**GO/NO-GO recommendation:** **GO with conditions** — fix test suite instability and pricing sync before next release; all other items are sprint-2+ work.

## Calibration Baseline (Phase 0)

1. **Stack:** React 18.3, TypeScript ~5.6, Vite 7.3.6, Tailwind CSS v3.4, Supabase (auth + 109 Edge Functions), deployed on Netlify
2. **TypeScript config:** `strict: false` — discriminated union narrowing and null-check rigor are not enforced project-wide. This is a deliberate choice for a prototype-stage product but limits type safety.
3. **Linting:** ESLint 9.x with `eslint-plugin-react-hooks` and `eslint-plugin-react-refresh`; 5 errors (all `prefer-const`), 0 warnings at audit time
4. **Testing:** Vitest 4.1 + jsdom 29 + Playwright for component/E2E; 152 test files (127 unit, 24 component), 778 tests total (712 pass, 66 fail in parallel)
5. **Security posture:** CSP hardened (no `unsafe-inline` in script-src), CORS wildcards eliminated, rate limiting on 68 Edge Functions, JWT server-side verification, DOMPurify sanitization
6. **Commercial guardrails:** `check:claim-boundaries` (418 files) and `check:commercial-source` scripts enforce language boundaries; `COMMERCIAL_SOURCE_OF_TRUTH.md` is the canonical commercial document
7. **Bundle optimization:** React.lazy() code splitting for 65+ routes, vendor manualChunks (React, Recharts, Radix, Supabase, PDF, Redoc); largest chunk is vendor-redoc at 1,187 KB (356 KB gzip)
8. **Payment integration:** Whop (primary), Stripe, Paddle — three payment providers with different entitlement models
9. **Pricing source of truth:** `src/lib/pricingCatalog.ts` — Free $0, Consumer Watchdog $9/mo, Professional $149/mo, Industrial $1,500/mo, Municipal $5,900/mo, Sovereign $2,500/mo

**Phase 0 exit criteria: Y** — all items met.

## Repo Map

**Purpose:** Canadian energy intelligence platform providing utility demand forecasting, regulatory proof packs, ESG/NPRI data workflows, and buyer-ready energy analytics.

**Entry points:**
- `src/main.tsx` — React root, global error handlers
- `src/App.tsx:1-361` — Router with 65+ lazy-loaded routes, Suspense fallbacks, error boundaries
- `supabase/functions/` — 109 Edge Functions (auth, data connectors, LLM, billing, cron jobs)

**Data flows:**
- Frontend → Supabase Edge Functions → External APIs (AESO, IESO, StatCan, CER, ECCC NPRI)
- Frontend → Supabase Auth → Whop/Stripe/Paddle for billing
- Frontend → Supabase DB (Postgres) for user data, scenarios, evidence packs

**Conventions:**
- Components: PascalCase `.tsx`, lazy-loaded via `React.lazy(() => import(...))`
- Lib: camelCase `.ts`, domain-organized (`utilityForecasting.ts`, `mlForecasting.ts`, etc.)
- Tests: `tests/unit/*.test.ts` (Vitest), `tests/component/*.spec.ts` (Playwright)
- Edge Functions: `supabase/functions/{name}/index.ts` with shared `_shared/cors.ts` and `_shared/rateLimit.ts`

## False Positive Log

1. **FP-1: "SensitivityResult not exported" (dismissed)** — Initially flagged as a missing export. Refuter verified that `SensitivityResult` was never an exported type; the component was using a stale reference. The fix (Phase 1a) replaced it with a local view model. This was a real bug, not a missing export.

2. **FP-2: "undici override breaks all tests" (downgraded)** — Initially flagged as a critical regression from the undici pnpm override. Refuter verified that only jsdom-dependent tests were affected, and the fix (scoping override to `<8.0.0`) resolved the issue. Downgraded from Critical to Low.
