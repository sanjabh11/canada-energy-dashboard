## Product Requirements

### PRODUCT #1: Indigenous Energy Intelligence Platform (PRIORITY P0)

**Objective:** Become THE data intelligence platform for Indigenous energy projects in Canada, serving communities, ICE Network, funders, and impact investors.

#### F1.1: Indigenous Project Registry (P0)

**User Stories:**
- As an Indigenous energy coordinator, I want to register my community's projects so that I can track them in one place and share them with potential partners.
- As an ICE Network member, I want to search for similar projects in other communities so that I can learn from their successes and avoid their mistakes.
- As a Wah-ila-toos funder, I want to see all funded projects in one dashboard so that I can track progress and impact.

**Acceptance Criteria:**
```gherkin
Given I am an Indigenous energy coordinator
When I submit a project to the registry
Then the project is added with fields:
  - Project name and description
  - Community name and location
  - Technology type (solar, wind, biomass, hydro, storage, etc.)
  - Capacity (kW or MW)
  - Status (planning, construction, operational, decommissioned)
  - Funding sources (Wah-ila-toos, CERRC, Northern REACHE, private)
  - Timeline (start date, completion date)
  - Budget and actual costs
  - Key partners (contractors, consultants, utilities)
  - UNDRIP/FPIC compliance status
And the project visibility is set to my preference:
  - Public (anyone can see)
  - ICE Network members only
  - Private (only my community)
And I can update the project at any time
And I can delete the project if needed
```

**Technical Specifications:**
- **Database:** Extend existing `indigenous_projects` table in Supabase
- **Fields:** 25+ metadata fields (see schema below)
- **Consent Management:** OCAP®-compliant consent workflow
- **API Endpoint:** `api-v2-indigenous-projects` (already exists, needs enhancement)
- **Storage:** PostgreSQL with Row-Level Security (RLS)
- **Backups:** Daily automated backups to S3

**Indigenous_projects Schema (Enhanced):**
```sql
CREATE TABLE indigenous_projects (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  
  -- Basic Info
  project_name TEXT NOT NULL,
  community_name TEXT NOT NULL,
  community_id UUID REFERENCES indigenous_communities(id),
  province TEXT NOT NULL,
  traditional_territory TEXT, -- e.g., "Cree Nation", "Anishinaabe"
  
  -- Project Details
  technology_type TEXT NOT NULL, -- solar, wind, biomass, hydro, storage, thermal
  capacity_kw DECIMAL,
  project_status TEXT NOT NULL, -- planning, construction, operational, decommissioned
  start_date DATE,
  operational_date DATE,
  decommission_date DATE,
  
  -- Financial
  total_budget DECIMAL,
  actual_cost DECIMAL,
  funding_sources JSONB, -- [{source: "Wah-ila-toos", amount: 500000}, ...]
  
  -- UNDRIP Compliance
  fpic_status TEXT, -- obtained, in-progress, not-applicable
  fpic_date DATE,
  undrip_compliant BOOLEAN DEFAULT false,
  consent_documentation_url TEXT,
  
  -- Impact Metrics
  jobs_created INT,
  emissions_avoided_tonnes_co2 DECIMAL,
  households_served INT,
  revenue_sharing_model TEXT,
  community_benefits JSONB,
  
  -- Partnerships
  lead_partners JSONB, -- [{name: "ABC Solar", role: "EPC contractor"}, ...]
  consultants JSONB,
  utility_interconnection TEXT,
  
  -- Data Sovereignty (OCAP®)
  visibility TEXT NOT NULL DEFAULT 'private', -- public, ice_network, private
  data_owner UUID NOT NULL REFERENCES edubiz_users(id),
  consent_expires_at TIMESTAMPTZ,
  
  -- Metadata
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  created_by UUID REFERENCES edubiz_users(id),
  updated_by UUID REFERENCES edubiz_users(id)
);

-- Indexes
CREATE INDEX idx_projects_community ON indigenous_projects(community_id);
CREATE INDEX idx_projects_status ON indigenous_projects(project_status);
CREATE INDEX idx_projects_tech ON indigenous_projects(technology_type);
CREATE INDEX idx_projects_visibility ON indigenous_projects(visibility);
```

**UI Components:**
- Project submission form (multi-step wizard)
- Project search/filter interface
- Project detail page
- Map visualization (integrated with Native Land Digital API)
- Bulk import (CSV upload for ICE Network)

**Build Effort:** 16 hours
- Form UI: 6 hours
- Backend API enhancements: 4 hours
- Search/filter: 4 hours
- Testing: 2 hours
